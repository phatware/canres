Detailed Description

Liquid Crystal Display Controller Adapter.

Macro Definition Documentation

#define LCDC_DELAY_US (   us)    LCDC_TAG_DELAY_US, ((us) & 0xFF), ((us >> 8) & 0xFF)

Predefined command sequence.

LCD controller adapter enables the creation and execution of pre-defined command sequences to facilitate the LCD initialization and other common LCD actions (i.e. enable or disable of the LCD). The commands supported can be divided in the following:

  • Delays
  • GPIO manipulation
  • MIPI DCS commands with / without parameters
  • JDI / Sharp type commands
  • External clock manipulation

To execute the command sequences, the function ad_lcdc_execute_cmds() must be called providing the memory space containing the commands and also the total size of the commands in bytes.

The following macros can be used to create the commands and their corresponding parameters.

Action Macro
Delay for the specified amount of microseconds LCDC_DELAY_US
Delay for the specified amount of milliseconds LCDC_DELAY_MS
Set the corresponding GPIO active LCDC_GPIO_SET_ACTIVE
Set the corresponding GPIO inactive LCDC_GPIO_SET_INACTIVE
Send a MIPI DCS command to the LCD LCDC_MIPI_CMD
Send a MIPI DCS parameter to the LCD LCDC_MIPI_DATA
Enable the LCD over MIPI DCS LCDC_MIPI_ENABLE
Disable the LCD over MIPI DCS LCDC_MIPI_DISABLE
Set color mode of the LCD over MIPI DCS LCDC_MIPI_SET_MODE
Set screen area that is being updated over MIPI DCS LCDC_MIPI_SET_POSITION
Set screen area that is active over MIPI DCS LCDC_MIPI_SET_PARTIAL
Perform a SW reset of the LCD over MIPI DCS LCDC_MIPI_SW_RST
Stop inversion mode of LCD over MIPI DCS LCDC_MIPI_EXIT_INVERT
Set the data mode of the LCD over MIPI DCS LCDC_MIPI_SET_ADDR_MODE
Set tear effect output sync of the LCD over MIPI DCS LCDC_MIPI_SET_TEAR_ON
Send a JDI / Sharp command to the LCD LCDC_JDI_CMD
Stop blinking of the LCD over JDI / Sharp interface LCDC_JDI_BLINKOFF
Start blinking black of the LCD over JDI / Sharp interface LCDC_JDI_BLINKBLACK
Start blinking white of the LCD over JDI / Sharp interface LCDC_JDI_BLINKWHITE
Start blinking inverted of the LCD over JDI / Sharp interface LCDC_JDI_BLINKINVERT
Clear the LCD over JDI / Sharp interface LCDC_JDI_CLEAR
Enable / disable the external clock generation LCDC_EXT_CLK_SET

Close LCD controller.

This function:

  • Aborts ongoing transactions
  • De-initializes the drivers associated with the controller
  • Resets controller interface IOs (as specified in ad_lcdc_open())
  • Releases the controller resources
Parameters

[in]forceForce close even if controller is busy
See also
ad_lcdc_open()
Returns
0: success, <0: error code
Warning
It is important to close the device when it is not needed since it will release the PLL clock so it can be turned off and save power.
int ad_lcdc_continuous_update_draw ( ad_lcdc_handle_t  handle,
const hw_lcdc_layer_t *  layer 
)

Perform a screen contents update with the layer parameters while in continuous update mode.

This function sets the layer parameters only. Frame generation is automatically done by the LCD controller while in continuous mode. This function is not blocking and it will return immediately.

Example usage:

{
ad_lcdc_handle_t hndl = ad_lcdc_open(LCDC_DEVICE);
if (hndl == NULL) {
// Handle open error
}
// Start continuous update when screen is on
ad_lcdc_continuous_update_start(hndl, frame_update_cb, &frame_update_cb_data);
while (1) {
// Block until previous frame has been generated
ad_lcdc_continuous_update_draw(hndl, layer, OS_EVENT_FOREVER);
...
}
// Stop continuous update when screen is off
ad_lcdc_continuous_update_stop(hndl);
dev_close(hndl);
}
Parameters
[in]handleHandle returned from ad_lcdc_open()
[in]layerLayer structure containing all layer information
See also
ad_lcdc_open()
ad_lcdc_close()
ad_lcdc_continuous_update_start()
ad_lcdc_continuous_update_stop()
Note
In continuous update mode, any change in the frame buffer will be depicted in the output of the controller and thus in the displayed contents. As a result, a rule of thumb would be to use a second frame buffer to prepare the frame contents and then use this function to display that buffer.
Any change in the layer settings will take effect after the end of the currently displayed frame (after frame end interrupt).
Warning
To synchronize with the frame generation, the callback function provided with the function ad_lcdc_continuous_update_start() should be used.
Returns
0 on success, <0: error
int ad_lcdc_continuous_update_start ( ad_lcdc_handle_t  handle,
ad_lcdc_user_cb  cb,
void *  user_data 
)

Initialize a continuous update of the LCD.

This function only sets up the continuous mode of the LCD update. In this mode, LCD controller continuously updates the LCD even if the contents of the frame remain constant. In order to update the screen contents, a callback function is provided by the application (with corresponding user data) which is called every time the frame should be updated.

Parameters
[in]handleHandle returned from ad_lcdc_open()
[in]cbCallback to call after transaction is over (from ISR context)
[in]user_dataUser data to pass to cb
See also
ad_lcdc_open()
ad_lcdc_close()
Note
The callback is called from within LCDC ISR.
Warning
The continuous update mode ends only with a call to ad_lcdc_continuous_update_stop() and this is when the resources are released.
Returns
0 on success, <0: error

Open LCDC controller.

  • Configures the controller interface IOs
  • Initializes the drivers associated with the controller
Parameters

Returns
!=0: handle that should be used in subsequent API calls, NULL: error
Note
The function will block until it acquires all controller resources
If required by the device's selected frequency (

LCDC_FREQ), this function tries to set the system clock to PLL96. Function ad_lcdc_close() has to be called to enable lowering the system clock frequency. If the clock is not successfully set, this function returns a NULL pointer indicating the failure.
