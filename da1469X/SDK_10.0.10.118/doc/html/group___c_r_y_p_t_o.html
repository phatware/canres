typedef void(* hw_crypto_cb) (unsigned int status)

Crypto engine callback.

This function type is used for a callback called when the AES/HASH engine generates an interrupt.

Parameters
[in]statusThe status register of AES/HASH at the time of the interrupt.

__STATIC_INLINE void hw_crypto_clear_pending_interrupt ( void  )

Clear pending interrupt from AES/HASH engine.

Note
This function clears the pending CRYPTO interrupt only on the NVI Controller. Use hw_aes_hash_clear_interrupt_req() to clear the source of CRYPTO interrupt on the AES/HASH engine.
void hw_crypto_enable_aes_hash_interrupt ( hw_crypto_cb  cb)

Enable interrupt for AES/HASH crypto engine.

Parameters
[in]cbA callback to be called when interrupt occurs. It must always be provided.
Note
AES/HASH engine and ECC engine are common sources of CRYPTO system interrupt. This function only enables CRYPTO interrupt itself and registers a callback for AES/HASH related CRYPTO interrupts. In order to fully enable AES/HASH interrupts hw_aes_hash_enable_interrupt_source() must also be called
