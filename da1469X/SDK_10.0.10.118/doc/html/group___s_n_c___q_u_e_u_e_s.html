)    _SNC_queues_snc_get_rq(rp, size, timestamp)

Function used in SNC context in order to obtain a read pointer on the CM33-to-SNC (read) queue as well as the size and the timestamp (if enabled) of the data to be read/popped.

Parameters
[out]rp(uint32_t*: use da() or ia()) the returned read pointer (NULL if the queue is empty)
[out]size(uint32_t*: use da() or ia()) the size of the data in bytes residing in the current queue position
[out]timestamp(uint32_t*: use da() or ia()) the timestamp accompanying the data (if enabled)

#define SNC_queues_snc_pop ( )    _SNC_queues_snc_pop()

Function used in SNC context in order to pop data from the CM33-to-SNC (read) queue.

A queue read pointer together with the data size and timestamp (if enabled), should first be obtained using the SNC_queues_snc_get_rq() function. After that, the user may read from the queue using that pointer. After reading the desired data, SNC_queues_snc_pop() should be called in order to update the contents of the queue.

#define SNC_queues_snc_push ( )    _SNC_queues_snc_push()

Function used in SNC context in order to push data into the SNC-to-CM33 (write) queue.

A queue write pointer should first be obtained using the SNC_queues_snc_get_wq() function. After that, the user may write inside the queue using that pointer. After writing the desired data, SNC_queues_snc_push() function has to be called in order to update the contents of the queue.

#define SNC_queues_snc_reset_wq ( )    _SNC_queues_snc_reset_wq()

Function used in SNC context in order to reset the SNC-to-CM33 (write) queue.

In essence, when calling this function, the chunks that have been marked as "written/pushed" in the SNC-to-CM33 (write) queue, are finally marked as free.
